A few smaller decisions

Pick the option you prefer in each row. Leave defaults if recommended is fine.

1. When a candidate is already in the cohort, how do we show it in the left column?

A

Show it in both columns, greyed/checked in the candidates column (recommended)

Easier to see "did I already pick this?" while scanning candidates. Click toggles either side.

B

Hide it from the candidates column once added

Candidates column shrinks to "what's still addable". Cleaner but you lose context if you remove from cohort.

2. What if cohort has experiments that don't match current filters? (e.g. you loaded a set, then narrowed filters)

A

Keep them in the cohort column with a small "outside filter" tag (recommended)

Cohort is independent of filters. The tag warns the user; they can still click to remove.

B

Auto-drop them when filters narrow

Cohort always equals selected ∩ filtered. Simpler invariant, but destructive if a filter change is exploratory.

3. Sort order in each column

A

By experiment title (recommended, matches sidebar today)

B

By date (newest first)

C

Sort dropdown (title / date / strain) on each column

4. Search within each column?

A

Yes — small search box at top of each column (recommended)

Useful when candidates list is hundreds long, and to verify a specific experiment is in the cohort.

B

No — rely on filters and scrolling
