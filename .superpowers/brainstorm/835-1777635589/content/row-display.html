Which variable tags should we show on each row?

Variables in this app are arbitrary key/value strings (e.g. "pulse size":"40", "feed rate":"0.2"). Each experiment can have many. We need to decide which ones get shown as tags inline.

A

Show only variables you've filtered on (recommended)

If your filters include "pulse size" and "feed rate", every row shows those two as small tags. Compact, relevant, and the tags answer the question "why did this row match my filter?".

Ferm 128-BO pulse size: 40 feed rate: 0.2 Strain X
B

Show all variables on each row

Every variable for every experiment becomes a tag. Most informative, but rows become wide/wrap heavily; harder to scan.

Ferm 128-BO pulse size: 40 feed rate: 0.2 induction: IPTG temp shift: 30C media: SD-Ura Strain X
C

Filtered variables inline + "more" expander for the rest

Filter-active variables show as tags. A small "+3" pill expands the row on click to reveal the remaining variables.

Ferm 128-BO pulse size: 40 feed rate: 0.2 +3 Strain X
